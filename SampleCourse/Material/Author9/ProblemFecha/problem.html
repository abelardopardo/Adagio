En este ejercicio se van a asentar los conceptos de programación orientada a objetos en Java, aplicándolos a una jeraquía de clases para definir figuras geométricas.

Importante: recuerda que es fundamental que vayas probando el código según vas implementando cada método, aunque no se pida explícitamente.

Apartado 1: Rectángulo

La primera clase a implementar representará un rectángulo y, cómo no, se llamará Rectangulo. Para caracterizar el rectángulo se utilizarán su base y su altura, que asumiremos que son números enteros. Por tanto, lo más adecuado es que el constructor tome como parámetros ambas variables.

Además del constructor y de los métodos accesores (get y set) básicos, la clase Rectangulo debe proporcionar los siguientes métodos: 

    /**
     * Devuelve una cadena indicando tipo de figura: "Rectangulo"
     */
    public String getTipoFigura() {
    }

    /**
     * Devuelve el área de la figura
     */
    public double calcularArea();

    /**
     * Representa en consola la figura
     */
    public void dibujarTxt();

El método dibujarTxt() debe dibujar la figura en la consola de texto. Por ejemplo, un rectángulo con base 6 y altura 3 se podría mostrar como:

Apartado 2: Cuadrado

Un cuadrado es un rectángulo con base y altura iguales. En Java, podemos crear una clase Cuadrado muy fácilmente, mediante herencia, a partir de la clase Rectangulo.

Programa la clase Cuadrado de tal forma que herede de la clase Rectangulo del apartado anterior. La clase Cuadrado sólo necesita un constructor que reciba un parámetro (el valor del lado), y que llame al constructor de Rectangulo con los parámetros base y altura iguales.

Probar esta nueva clase añadiendo el código necesario a la clase PruebaFiguras.

Observa que gracias a la herencia, pueden invocarse los métodos programados en la clase Rectangulo sobre un objeto de tipo Cuadrado, sin necesidad de reprogramarlos.

¿Qué ocurre al invocar al método getTipoFigura sobre un objeto de tipo Cuadrado? Para solucionarlo, reprograma en la clase Cuadrado el método getTipoFigura de modo que devuelva el valor correcto.

  • ¿En qué consiste la herencia?
  • ¿Cómo se indica en Java que una clase hereda de otra?
  • ¿Qué métodos de la clase padre son visibles desde la clase hija?
  • ¿En qué consiste la sobreescritura (override) de métodos?
  • Recuerda que a diferencia del resto de métodos, las clases hijas no heredan automáticamente los constructores de la clase padre, pero pueden invocarse con ayuda de super().
